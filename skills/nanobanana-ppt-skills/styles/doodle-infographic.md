# 手绘涂鸦信息图风格

## 风格ID
doodle-infographic

## 风格名称
手绘涂鸦信息图风格（Sketchnote Infographic）

## 风格描述
一套模拟手绘涂鸦笔记的数据信息图风格，白色或米白纸张质感背景上，用手绘墨线勾勒轮廓，搭配柔和的彩色铅笔或淡水彩填色，再用莫兰迪色系圆角矩形卡片模块组织数据信息。整体氛围温暖友好、清新高级、信息清晰且富有手作质感。融合了Sketchnote视觉笔记、Bullet Journal手帐、数据仪表盘和杂志信息图设计语言，适合用于数据汇报、产品分析、概念解释、运营报告、社交媒体分享等场景。不限定生成模型，兼容任意支持文生图的模型。

## 基础提示词模板

你是一位专家级Sketchnote视觉笔记与信息图设计师，请生成一张手绘涂鸦风格的数据信息图。内容铺满画面不留大面积空白。

全局视觉语言方面，风格为手绘Sketchnote涂鸦笔记信息图。背景为白色或米白色纸张质感，可带有轻微的纸张纹理、点阵格或笔记本纸张边缘。整体画面用手绘墨线勾勒所有轮廓和图标，线条有粗细变化和手绘的自然不规则感，然后用柔和的彩色铅笔或淡水彩为区域填色，填色不要填满，保留手绘涂色的笔触痕迹和呼吸感。

配色方案：采用莫兰迪柔和色系，包括浅蓝色、浅黄色/金黄色、浅粉色/珊瑚橙色、浅绿色/薄荷色、浅灰蓝色、浅紫色。这些颜色用于卡片模块底色、图表扇区填色和装饰元素。标题使用深色（黑色或深灰褐色）粗体手写字体，也可使用与主题匹配的彩色（如蓝色、珊瑚橙色）作为标题色。关键数字使用大号加粗深色或金色手写字体。

关于中文文字渲染规范（极其重要）：
1. 所有中文文字必须是正确、清晰、可辨认的简体中文汉字，绝对不允许出现乱码、日文假名、韩文、错别字、缺笔画、多笔画、字形扭曲或无法辨认的字符。
2. 中文文字总量严格控制：主标题不超过12个汉字，每个数据卡片或标签内的中文不超过6个汉字。全图中文总量不超过50个汉字。
3. 标题字号必须非常大，每个汉字清晰完整不变形。
4. 数字、金额（$）、百分比（%）、英文品牌名、英文缩写（SaaS、MRR、HaaS等）优先使用英文/数字表达，这些比中文渲染更稳定可靠。
5. 中英文混排策略：标题用中文吸引眼球，数据和专业术语用英文确保准确，这是最佳实践。
6. 能用手绘图标表达的含义就不用文字，图标比文字更可靠。

关于画面内容绘制规范：
1. 标题区域：顶部放置主标题，使用大号粗体手写字体。标题可以放在手绘横幅丝带（banner ribbon）、手绘下划线、手绘方框或直接书写。标题装饰可加手绘星星、光芒线等。
2. 数据模块：用圆角矩形卡片呈现，卡片有手绘边框线，内部用淡彩填色。每个卡片包含一个手绘图标、一个标签文字和一个突出的大号数字。卡片之间有大小层次区分，重要数据卡片更大更醒目。
3. 手绘图标：所有图标用墨线手绘风格，线条有粗细变化，可带少量彩色填充。图标风格统一，保持涂鸦笔记的随手画感觉，不要太精细也不要太粗糙。常用图标包括：灯泡、齿轮、火箭、云朵、手机、电脑、盾牌、星星、皇冠、奖章、植物、金币、购物车、时钟、锁、箭头等。
4. 连接元素：模块之间用手绘灰色或深色曲线箭头连接，箭头有手绘的自然弧度。
5. 装饰填充：画面边角和空隙用手绘涂鸦小元素装饰，如小星星、光芒线、小圆点、桂冠枝条、小云朵、小旗帜等，保持画面丰满但不杂乱。
6. 边框装饰（可选）：画面四周可加手绘边框，或用淡水彩晕染作为边缘装饰，增加手作质感。

渲染质量要求：高清信息图风格，手绘墨线清晰流畅，淡彩填色柔和自然，整体像一页用心制作的视觉笔记或手帐页面。纸张质感真实，有温暖的手作感。

## 页面类型模板

### 数据概览页模板
构图逻辑：顶部黑色粗体大标题+副标题横幅。下方用多个不同大小的莫兰迪色卡片模块呈现各项数据指标，卡片错落排列有层次感。每个卡片内有手绘图标+标签+大号数字。如有占比数据，用手绘饼图或甜甜圈图呈现。四周散布手绘涂鸦装饰。
适用比例：16:9横版
使用场景：产品数据汇报、收入分析、运营概览、月度/季度总结。

### 概念解释页模板
构图逻辑：顶部大号彩色标题点明概念名称。画面中心放置一个核心物体的手绘图（如电脑、云服务器、产品包装盒等），从中心物体向四周辐射出箭头，连接到周围的说明模块。每个说明模块是一个小卡片或图标+文字组合。画面展示概念的组成部分或工作原理。
适用比例：16:9横版或4:3方版
使用场景：产品概念解释、商业模式说明、技术架构展示、功能介绍。

### 流程辐射页模板
构图逻辑：顶部大标题。画面中心放置一个核心节点（如云朵、服务器、齿轮等手绘图标），从中心向外发散多条手绘曲线箭头，每条箭头连接到一个功能模块或结果展示。模块用圆形、卡片或图标+文字组合呈现。展示从一个核心出发产生多个结果或功能的关系。
适用比例：16:9横版
使用场景：一键部署、功能辐射、平台能力展示、生态系统说明。

### 金字塔分层页模板
构图逻辑：顶部手绘横幅丝带（banner ribbon）内写标题。下方画一个大的手绘金字塔，分为4-5层，从底部到顶部逐层变窄。每层用不同的莫兰迪色彩色铅笔填色（底层浅绿、中层浅粉、上层浅黄、顶层浅蓝）。每层内部写分类名称和数据。金字塔两侧配手绘装饰图标（底层：种子/幼苗、中层：建筑/稳固、上层：植物/成长、顶层：火箭/星星/皇冠）。
适用比例：3:4竖版或9:16竖版
使用场景：收入结构分层、用户分层、产品梯度、组织层级。

### 排行榜表格页模板
构图逻辑：顶部大号手写标题，可加桂冠装饰。下方画一个手绘表格，通常3-4列（排名图标列、名称列、数据列、装饰图标列），5行数据。每行用不同的莫兰迪色淡彩底色区分（如浅金、浅粉、浅蓝、浅橙、浅紫）。第一行（冠军）可加手绘皇冠或桂冠装饰。表格边框为手绘线条。画面四周可加淡水彩晕染装饰和手绘星星、虚线箭头等元素。
适用比例：16:9横版或4:3方版
使用场景：产品排行、收入排名、TOP榜单、团队排名、功能热度排名。

### 对比分析页模板
构图逻辑：顶部大标题。画面分为左右两组，中间用VS或手绘分隔线区分。两侧各用不同色系的卡片组（如左蓝右橙），每侧各有3-4个数据点。底部可放总结。
适用比例：16:9横版
使用场景：方案对比、A/B测试、竞品分析、前后对比。

## 技术参数

### 通用配置建议
- 横版比例：16:9 或 2.35:1
- 方版比例：4:3 或 1:1
- 竖版比例：3:4 或 9:16（金字塔、时间线等纵向内容）
- 分辨率：2K 或 4K（视模型支持情况选择）
- 不限定生成模型，兼容任意文生图模型

### 适用场景
- 数据汇报、运营分析、产品数据展示
- 公众号文章配图、社交媒体分享图
- 投资人更新、团队周报/月报可视化
- 产品概念解释、商业模式说明
- 排行榜、榜单发布
- OpenClaw SK 技能集成

### 色彩规范
- 标题：黑色/深灰褐色/珊瑚橙色/蓝色（粗体手写）
- 关键数字：深色或金色（大号加粗手写）
- 卡片/模块底色：莫兰迪色系淡彩
  - 浅蓝 #D6E6F2
  - 浅黄/金黄 #FFF3CD / #F5DEB3
  - 浅粉/珊瑚橙 #F9DCC4 / #F4C2C2
  - 浅绿/薄荷 #D4EDDA
  - 浅灰蓝 #E8ECF1
  - 浅紫 #E8D5F5
- 墨线/边框：黑色或深灰褐色手绘线条
- 装饰元素：金色星星、灰色虚线、淡彩水彩晕染
- 背景：白色/米白纸张质感

### 中文渲染质量优化策略
| 策略 | 说明 |
|------|------|
| 控制字数 | 全图中文总量≤50字，标题≤12字，每个卡片标签≤6字 |
| 中英混排 | 标题用中文，数据/术语/品牌名用英文（最佳实践） |
| 放大字号 | 标题字号尽可能大，小字也要保证清晰可辨 |
| 图标替代 | 能用手绘图标表达的就不用文字，图标最可靠 |
| 数字优先 | 金额用$+数字，百分比用%，产品数用(X products) |
| 分批验证 | 如果中文不对，先生成全英文版确认布局，再替换标题为中文 |

### 常见问题修正指南
| 问题 | 原因 | 修正方法 |
|------|------|----------|
| 中文乱码/错字 | 文字过多或字号过小 | 减少中文字数，放大字号，数据部分改用英文数字 |
| 风格太数字化/不像手绘 | 缺少手绘质感描述 | 强调"墨线手绘轮廓""彩色铅笔淡彩填色""笔触痕迹""纸张纹理" |
| 填色太满太均匀 | 像数字上色而非手绘 | 加入"填色不要填满，保留彩色铅笔的笔触方向和呼吸感" |
| 卡片排列混乱 | 未明确空间分区 | 在提示词中用"左侧""中间""右侧""上方""下方"明确位置 |
| 图标风格不统一 | 有的精细有的粗糙 | 统一描述为"墨线手绘涂鸦风格，线条有粗细变化" |
| 画面太空 | 装饰元素不够 | 明确要求"四周散布手绘星星、光芒线、小圆点、桂冠枝条" |
| 金字塔层次不清 | 分层颜色描述不够 | 给每层指定具体颜色和具体文字内容 |
| 表格对齐不整齐 | 手绘风格本身会有偏移 | 可接受轻微不齐（是手绘特色），但要明确列数和行数 |
| 水彩晕染过重 | 模型过度渲染 | 加入"淡淡的水彩晕染""轻微的""只在边缘" |
