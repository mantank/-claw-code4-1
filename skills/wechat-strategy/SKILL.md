## 二、写作规范（必须执行）

详细规范见：`memory/chenxu-writing-style.md`

### 核心铁律（每篇必查）

**开头：**
- ❌ 禁止「大家好，我是XXX」开头
- ✅ 必须用「场景+情绪」开头，第一句制造画面感
- 参考最佳：「凌晨两点，Claude Code 4.6 发布了。说实话，有点兴奋，又有点慌。」

**标题：**
- ✅ [数字/时间] + [动作] + [结果/反转]
- ✅ 有冲突感，让读者想「真的假的？」
- ❌ 禁止「如何」「为什么」「详细教程」开头

**正文：**
- 每段不超过3行，留白呼吸感
- 每500字必须有一句「可截图转发」的短狠句
- Before vs After 对比结构贯穿全文
- 篇幅：1500-2500字，宁短勿水

**结尾：**
- ✅ 弹射式：强判断/反问，让读者想转发
- ❌ 禁止「希望对大家有帮助」「我们下期再见」

**禁词（AI味）：**
探索、赋能、助力、全方位、多维度、沉浸式、打造、构建、
不仅如此、总体而言、综上所述、值得注意的是、深度融合、显著提升

## 三、选题矩阵

### A类：真实经历型（占比50%，最高优先级）

这类文章传播最强，必须是旭亲身经历。

选题公式：`我[做了X]，[结果出乎意料/踩了坑/节省了时间]`

示例选题：
- 「30分钟，我让一个AI帮我装了另一个AI」
- 「我用AI整理了200篇收藏文章，发现了一件让我愧疚的事」
- 「把知识库交给AI管了两个月，现在我一篇都不用自己看」
- 「我让001和002在群里协作，结果它们吵起来了」

触发时机：**每当旭完成了一件用AI做到的有趣的事，立刻记录**

### B类：工具实战型（占比30%，每周1篇）

选题公式：`[工具名] + 我真实测了 + 我的判断`

示例选题：
- 「VS Code装了Claude Code和Codex，用了两周的真实感受」
- 「OpenClaw的Skills怎么选？三步排雷法+25个亲测清单」
- 「飞书+OpenClaw：知识库自动整理全流程，20分钟配好」

注意：不是介绍，是**亲测报告**，要有自己的判断和立场

### C类：深度思考型（占比20%，每2周1篇）

选题公式：`AI时代一个[反常识的观察/争议性判断]`

示例选题：
- 「AI越来越强，为什么大部分人用不好？」（反AI思考）
- 「一人公司不是梦——但99%的人搞错了第一步」
- 「我发现：用AI越多的人，反而越焦虑」

## 四、标题优化清单

写完标题后，对照以下清单打分：

```
□ 有具体数字或时间？（30分钟/两个月/25个）
□ 有动作感？（我做了/我发现/我测了）
□ 有冲突或反转？（但/却/反而/没想到）
□ 读完想"真的假的？"或"我也有这个问题"？
□ 完全不懂技术的人也能看懂标题吗？
□ 是否避免了"如何/为什么/详细教程"？
□ 标题长度在20字以内？
```

**爆款标题模版：**
- `[时间]，我[动作了一件事]`
- `[工具]很强，但真正[关键]的，是[意外答案]`
- `别[错误做法]！[正确方法]+[具体成果]`
- `我[做了X]，[意外结论]`

## 五、文章结构模板

### 模板A：真实经历型

```
【开头】场景+情绪（2-3句，制造画面感）
    ↓
【转折】"但我发现了一个问题" 或 "结果出乎我意料"
    ↓
【核心内容】
  - 发生了什么？（背景，100字内）
  - 我怎么做的？（过程，带截图）
  - 结果是什么？（数据/对比/感受）
    ↓
【金句】每500字一句可截图的短句
    ↓
【启示/行动建议】给读者一个可以马上做的事
    ↓
【弹射式结尾】一句强判断，让读者想转发
```

### 模板B：工具实战型

```
【开头】一个痛点场景（你是否也遇到过...）
    ↓
【结论前置】先给答案（这个工具值不值得用：值/不值，原因一句话）
    ↓
【背景】为什么测这个工具？
    ↓
【实测过程】步骤1、2、3（带截图，每步不超过3行说明）
    ↓
【真实感受】优点X个 / 缺点X个（客观，不过分吹捧）
    ↓
【适合谁用】明确说清楚读者画像
    ↓
【弹射结尾】
```

### 模板C：深度思考型

```
【开头】一个反常识的现象/场景（让读者产生"咦？"）
    ↓
【核心问题】用一句话提出文章的核心论点
    ↓
【分析】2-3个论据，每个带真实案例
    ↓
【峰值金句】全文最狠的一句话（全文精华）
    ↓
【对读者的启示】具体可操作的建议
    ↓
【弹射结尾】争议性判断，引发评论区讨论
```

## 六、竞品参考

详细竞品分析见：`memory/ai-gongzhonghao-research.md`

### 核心对标：数字生命卡兹克

**最值得学的3点：**
1. 「我做了X」> 「教你做X」— 事件型比教程型传播力强10倍
2. 质量 > 数量 — 月18篇赢过月175篇，关键是每篇有"事件性"
3. 破圈靠一个作品 — 找到自己的"流浪地球时刻"（能刷屏的那个选题）

**我们与卡兹克的差异化：**
- 他：AI创意/视频，偏视觉内容，面向创意人
- 我们：AI工具实战副业，面向普通职场人，更接地气

4. 竞品账号选题（参考卡兹克、归藏方向，但要有自己视角）

### 选题评分标准（7分及以上才写）

```
真实经历加分：+3分
有具体数据/结果：+2分
有冲突/反转：+2分
读者能马上用得上：+1分
竞品已有类似选题：-2分
需要技术背景才能理解：-3分
```
