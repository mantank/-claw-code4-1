# CONTENT-PIPELINE.md — 公众号文章自动化流水线

> 目标：95%自动化，旭只做最终审核和发布。
> 参与者：001（编辑/质控）→ 002（写手/配图）→ 旭（审核/发布）

---

## 流水线全流程

```
每天14:30 Cron触发001
    ↓
【第1步】001选题（从当天选题情报中挑1个最佳）
    ↓
【第2步】001写选题Brief → 发给002
    ↓
【第3步】002写大纲 → 存文件
    ↓
【第4步】001审核大纲 → 给修改意见或通过
    ↓
【第5步】002写初稿v1 → 存文件
    ↓
【第6步】001审核初稿 → 打分 + 具体修改意见
    ↓
【第7步】002修改 → v2（最多迭代3轮）
    ↓
【第8步】001终审通过 → 通知002配图
    ↓
【第9步】002生成封面图 + 内容配图（白板马克笔风格）
    ↓
【第10步】001打包成果 → 发给旭审核
    ↓
【旭】确认OK → 002推送草稿箱 → 旭手动发布
```

---

## 各步骤详细规则

### 第1步：选题（001）
- 数据来源：当天的选题情报cron输出 + X热点 + 行业动态
- 选题标准（参考 THESIS.md）：
  - 普通人能看懂、能用上
  - 有热度或有争议性
  - 跟AI/Agent/OpenClaw相关
  - 不重复最近7天写过的话题
- 输出：1个选题 + 选题理由（3句话）

### 第2步：选题Brief（001→002）
- 写入路径：`/root/.openclaw-002/workspace/pipeline/brief.md`
- Brief包含：
  - 选题标题（建议）
  - 目标读者（谁会看这篇）
  - 核心观点（这篇文章要传递什么）
  - 参考素材链接（如果有）
  - 字数要求：1500-2500字
  - 风格要求：口语化、短句、有观点、不端着
- 通知方式：sessions_send 给002

### 第3步：大纲（002）
- 写入路径：`/root/.openclaw-002/workspace/pipeline/outline.md`
- 大纲格式：标题 + 各段落要点 + 结尾钩子
- 002完成后通知001

### 第4步：大纲审核（001）
- 审核标准：
  - 逻辑通不通
  - 有没有偏离Brief方向
  - 结构是否适合公众号阅读（短段落、有节奏）
- 通过 → 通知002写初稿
- 不通过 → 写具体修改意见到 `pipeline/outline-feedback.md`，通知002改

### 第5步：初稿v1（002）
- 写入路径：`/root/.openclaw-002/workspace/pipeline/draft-v1.md`
- 必须遵守：
  - FEEDBACK-LOG.md 里的写作规则
  - 结尾用 templates/article-endings.md 模板
  - 不用破折号、不用表情包堆砌
- 完成后通知001

### 第6步：初稿审核（001）
- 打分：0-100分
- 审核维度：
  - 标题吸引力（≤20字）
  - 开头钩子（前3句能不能留住人）
  - 核心观点是否清晰
  - 语气是否像人话（不像AI写的）
  - 结尾是否有行动引导
- 80分以上 → 直接进配图环节
- 60-79分 → 给修改意见，002改v2
- 60分以下 → 重写（极少发生）

### 第7步：迭代（最多3轮）
- 001每轮给具体的修改意见（不是"写好一点"，而是"第3段太长，砍一半"）
- 002每轮存新版本：draft-v2.md、draft-v3.md
- 3轮后不管多少分都进配图环节（避免死循环）

### 第8步：终审通过（001）
- 确认文章最终版本
- 写入：`pipeline/final.md`（最终定稿）
- 通知002进入配图环节

### 第9步：配图（002）
- 封面图：1张，白板马克笔风格（nanobanana-ppt-skills）
- 内容配图：根据文章段落生成6-10张PPT风格配图
- 图片存入：`pipeline/images/`
- 完成后通知001

### 第10步：打包交付（001→旭）
- 001发Telegram消息给旭，包含：
  - 文章标题
  - 最终评分
  - 文章摘要（3句话）
  - 封面图预览
  - "回复OK我安排发布，回复修改意见我让002改"
- 旭确认OK → 001通知002推送草稿箱
- 旭要改 → 001转达修改意见给002

### 核心原则（2026-03-10 旭确认）
- 001自动审稿、自动给002修改意见、自动循环，不打扰旭
- 最多迭代到v3，v3后不管分数都进配图
- 只有最终成品（文章+配图）才推给旭确认
- 旭说OK才发布，旭说改就继续改
- 全文禁止emoji、禁止字数统计

- 001收到cron触发 → 读选题情报 → 开始流水线
- 预计2-3小时跑完全流程
- 17:00前旭收到成品审核

## 质量红线（001审核时必须检查）

1. ❌ AI味太重（"在当今快速发展的时代..."）→ 打回重写
2. ❌ 标题超过20字 → 打回改标题
3. ❌ 没有结尾钩子 → 打回加结尾
4. ❌ 段落超过4行 → 打回拆段
5. ❌ 用了破折号 → 打回替换
6. ❌ 观点模糊（"这取决于..."）→ 打回给明确观点
