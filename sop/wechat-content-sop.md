# 公众号内容生产SOP v1.0

> 目标：从选题到发布全流程标准化，002执行80%，001润色，老大只做拍板和发布。
> 公众号：深夜开发者LND

---

## 流程总览

```
选题调研(002) → 老大拍板 → 大纲(002) → 初稿(002) → 润色(001) → 老大审核 → 配图(002) → 排版(002) → 老大终审发布
```

预计周期：1篇文章 = 2-3天（含审核等待时间）

---

## 第一步：选题调研（002执行）

### 输入
- 老大给方向关键词，或002自主调研

### 执行动作
1. **搜热点**：Brave Search搜近7天相关热词（AI工具/效率/副业/一人公司）
2. **看竞品**：搜3-5篇同类公众号的近期爆文标题
3. **查数据**：往期文章阅读数据（Notion公众号文章库）

### 输出格式
```
## 选题推荐（3选1）

### 选题A：[标题草案]
- 热度依据：xxx
- 目标读者：xxx
- 核心卖点：一句话说清为什么有人想看
- 预估难度：⭐~⭐⭐⭐

### 选题B：...
### 选题C：...
```

### 质量标准
- 每个选题必须有热度依据（不能凭感觉）
- 标题要有画面感+好奇心+利益点
- 至少1个选题跟"普通人用AI"相关（品牌主线）

### 卡点 → 老大拍板选哪个

---

## 第二步：写大纲（002执行）

### 输入
- 老大选定的选题
- 风格指南（templates/writing-style-guide.md）

### 执行动作
按以下结构输出大纲：

```
## 大纲：[文章标题]

### 开头钩子（1段）
- 用什么故事/问题/场景开头
- 预计50-100字

### 核心模块（3-5个）
- 模块1：[小标题] — 要讲什么，核心观点是什么
- 模块2：...
- 模块3：...

### 结尾（1段）
- 总结/金句/行动号召
- 引流钩子（加微信送xxx）

### 配图规划
- 需要几张图
- 每张图的位置和内容描述
```

### 质量标准
- 总篇幅控制在1500-2500字（手机阅读友好）
- 每个模块有明确观点，不是流水账
- 开头钩子必须3秒抓住人

### 卡点 → 老大看大纲是否需要调整（可选，不满意才介入）

---

## 第三步：写初稿（002执行）

### 输入
- 确认的大纲
- 风格指南
- 3篇范文参考（articles/目录下已发布文章）

### 执行规则

**必须遵守：**
- 每段不超过2行（手机端2行=约40-50字）
- 一屏 = 2/3文字 + 留白/图片位置
- 用"说人话就是"翻译所有技术概念
- 禁止AI味词汇：赋能、痛点、抓手、底层逻辑、范式、深度赋能、一站式、全方位
- 禁止开头用"在当今xxx时代"、"随着xxx的发展"
- 必须有至少1个真实案例/截图位置
- 结尾必须有品牌slogan："我先踩坑，你少走弯路。"
- 结尾必须有引流钩子

**语气检查清单：**
- [ ] 读起来像朋友聊天，不像课堂讲课？
- [ ] 有没有自嘲或吐槽的地方？
- [ ] 技术部分有没有"说人话"翻译？
- [ ] 段落够短吗？有没有超过3行的大块文字？

### 输出
- 完整markdown文稿
- 标注配图位置：📷【配图x：描述】

### 卡点 → 交给001润色

---

## 第四步：润色（001执行）

### 输入
- 002的初稿

### 执行动作
1. **去AI味**：对照humanizer-zh规则，改掉机械化表达
2. **加人味**：补充口语化表达、节奏感、适当情绪
3. **改标题**：确保标题有搜索关键词+好奇心驱动
4. **检查结构**：段落长度、配图密度、信息节奏
5. **补细节**：如果有老大的真实经历可以加，加进去

### 质量标准
- 读3遍，每遍检查不同维度（内容→语气→排版）
- 最终稿读起来不能像AI写的
- 标题至少准备3个备选

### 输出
- 润色后的终稿（保存到articles/）
- 3个备选标题
- 推到Notion公众号文章库（草稿状态）

### 卡点 → 老大审核终稿

---

## 第五步：配图（002执行）

### 输入
- 终稿中的📷标注

### 执行动作
1. 根据描述生成配图（优先级：即梦AI > 万相2.6 > Gemini）
2. 封面图单独做（卡通插画风，匹配品牌形象）
3. 文中插图保持风格统一

### 图片规格
- 封面图：横版 900x383 或 2.35:1
- 文中图：竖版或方形，宽度至少750px
- 风格：卡通插画 / 像素风（跟往期保持一致）
- 中文文字不放进AI生图里（AI渲染中文都很烂），文字用Canva叠加

### 输出
- 所有图片文件（保存到 images/article-xx/）
- 封面图单独标注

---

## 第六步：排版（002执行）

### 输入
- 终稿 + 配图

### 执行动作
1. 转换为Markdown Nice兼容格式
2. 插入图片位置标记
3. 检查排版要素：
   - 小标题用 ## 
   - 重点加粗
   - 引用框用 >
   - 代码块用 ```
   - 列表对齐

### 输出
- 可直接粘贴到 editor.mdnice.com 的markdown文本
- 配图文件按顺序命名

### 卡点 → 老大终审

---

## 第七步：发布（老大执行）

### 动作
1. 打开 editor.mdnice.com，粘贴markdown
2. 手动插入图片
3. 预览检查
4. 复制到公众号编辑器
5. 设置封面图、摘要、原创标记
6. 发布
7. 更新Notion状态为"已发布"，填写发布日期

### 发布后
- 002负责：5渠道分发（小红书/知乎/即刻/V2EX/朋友圈文案）
- 001负责：48h后统计阅读数据，更新Notion
