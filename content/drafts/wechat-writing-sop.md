# 公众号写作SOP v1

## 目标
从选题到发布，2小时内完成一篇公众号文章。

---

## 流程（6步）

### Step 1：选题确认（10分钟）
**谁做：** 老大提方向 / 零零壹建议
**产出：** 一句话选题 + 目标读者

选题检查清单：
- [ ] 标题能让不懂技术的人想点进来？
- [ ] 有真实经历/案例支撑？
- [ ] 和"普通人用好AI"这个定位相关？
- [ ] 不和前几篇重复？

标题原则（血泪教训）：
- ✅ 有画面感："我让AI接管了我的知识库"
- ✅ 有好奇心："它比我自己整理得还好"
- ✅ 有利益点："花20分钟搭好"
- ❌ 纯技术："OpenClaw配置飞书API教程"

### Step 2：大纲（15分钟）
**谁做：** 零零壹出初版 → 老大调整
**产出：** 6段结构大纲

推荐结构（万能模板）：
1. 痛点场景（引共鸣）
2. 转折（我的解决方案）
3. 效果展示（前后对比，勾住读者）
4. 完整教程/步骤（核心价值）
5. 为什么有效（用完再讲道理）
6. 行动号召（今晚就开始）

### Step 3：正文（40分钟）
**谁做：** 零零壹写初稿 → 写入Notion
**产出：** 完整正文（2500-3500字）

写作规范：
- 每段不超过3行（手机阅读友好）
- 加粗关键句（扫读也能抓到重点）
- 教程步骤编号清晰，每步配截图位置标注
- 口语化 > 书面化
- 用"你"不用"大家"

### Step 4：截图配图（30分钟）
**谁做：** 老大截图 + 零零壹生成封面图
**产出：** 文内截图 + 封面图

截图规范：
- 标注关键操作区域（红框/箭头）
- 模糊敏感信息（API Key等）
- 截图尺寸统一

封面图：
- 风格：卡通插画（匹配数字形象）
- 工具：ChatGPT生成 / Evolink Z-Image
- 尺寸：公众号封面比例 2.35:1（900x383）

### Step 5：排版（15分钟）
**谁做：** 老大在Markdown Nice排版
**产出：** 排版后的文章

排版流程：
1. 从Notion复制正文
2. 打开 editor.mdnice.com
3. 粘贴 → 选主题 → 插入图片
4. 预览确认 → 复制到公众号编辑器

### Step 6：发布（10分钟）
**谁做：** 老大发布
**产出：** 已发布文章

发布检查清单：
- [ ] 标题有吸引力？（找2-3个备选标题）
- [ ] 摘要写了？（显示在分享卡片上）
- [ ] 封面图上传了？
- [ ] 文内图片都加载正常？
- [ ] 原创标签勾选了？
- [ ] 底部引导关注了？

---

## 时间分配总结

| 步骤 | 时间 | 负责人 |
|------|------|--------|
| 选题确认 | 10min | 老大+零零壹 |
| 大纲 | 15min | 零零壹为主 |
| 正文 | 40min | 零零壹为主 |
| 截图配图 | 30min | 老大为主 |
| 排版 | 15min | 老大 |
| 发布 | 10min | 老大 |
| **合计** | **2小时** | |

---

## 优化方向（后续迭代）
- 封面图模板化（固定风格+标题区域，换文字就行）
- 排版模板固定（Markdown Nice存好主题，不用每次选）
- 建立选题库（平时有灵感就记下来，写的时候不用从零想）
- 发布后数据追踪（阅读量/转发/新关注，找到什么类型最受欢迎）
