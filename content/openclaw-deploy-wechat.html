一个非程序员的 OpenClaw 部署踩坑实录

我不是程序员。

但昨晚花了大概三个小时，把自己的 AI 助手部署到了 Telegram 上。中间踩了不少坑，全记下来了，希望能帮到后来人。

OpenClaw 是什么？
一个开源框架，把 Claude、GPT 这些大模型接到 Telegram、WhatsApp 上。装好之后，你就有了一个 24 小时在线的私人 AI 助手，手机上随时随地跟它聊。

📋 部署前准备

我用的是腾讯云 OpenCloudOS 服务器。你需要提前准备：

  • 一台 Linux 服务器（CentOS / Ubuntu / OpenCloudOS 都行）
  • Anthropic API Key（去 console.anthropic.com 申请）
  • Telegram Bot Token（通过 @BotFather 创建）
  • 你的 Telegram 用户 ID（用 @userinfobot 查）

🕳️ 坑 1：Node.js 装不上

OpenClaw 跑在 Node.js 上，得先装。

官方脚本跑到一半直接报错了，说不支持当前系统。我的 OpenCloudOS 明明是 RPM 系的，但它就是不认。

解决：别用官方脚本，手动来。

# 方法一：直接装
dnf install -y nodejs npm

# 方法二：用 nvm（推荐，兼容性更好）
curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.40.1/install.sh | bash
source ~/.bashrc
nvm install --lts

🕳️ 坑 2：缺 Git

npm 装 OpenClaw 时报错：npm error path git errno -2

系统没自带 git，一行搞定：

dnf install -y git

⚙️ 安装和配置

# 安装
npm install -g openclaw@latest

# 初始化配置
openclaw onboard

会弹出交互式菜单，主要填四个东西：

  1. 模型和 API Key — 选模型，填 Anthropic 的 Key
  2. Telegram Bot Token — BotFather 给你的那串
  3. Telegram 用户 ID — @userinfobot 查到的数字
  4. 工作目录 — 默认就行

💡 模型怎么选？

建议：先拿 Sonnet 4.5 跑通流程，确认没问题再换 Opus。不是所有 API Key 都有权限用 Opus。

模型 价格（每百万token） 体验
Opus 4.6 $5 / $25 最强，需高等级账户
Opus 4.5 $5 / $25 旗舰，性价比最高
Sonnet 4.5 $3 / $15 日常够用，推荐起步

🕳️ 坑 3：API Key 404

配好之后给 Bot 发消息，回了个 HTTP 404: not_found_error。

折腾了半天，发现是 Key 的问题。用 curl 直接测最快：

curl -X POST https://api.anthropic.com/v1/messages \
  -H "x-api-key: 你的KEY" \
  -H "content-type: application/json" \
  -H "anthropic-version: 2023-06-01" \
  -d '{"model":"claude-sonnet-4-5-20250929",
       "max_tokens":100,
       "messages":[{"role":"user","content":"hello"}]}'

返回 authentication_error 就是 Key 有问题，去 console.anthropic.com 重新生成。

🔗 配对 Telegram

第一次给 Bot 发消息，它会回一串配对码：

Pairing code: KW39NZUZ

回到服务器上跑：

openclaw pairing approve telegram KW39NZUZ

搞定。之后就能正常对话了。

📌 常用命令速查

openclaw status          # 看运行状态
openclaw gateway restart # 重启
openclaw logs --follow   # 实时日志
openclaw doctor          # 出问题先跑这个
npm update -g openclaw   # 更新版本

总结

整个过程顺利的话 30 分钟，算上踩坑两三个小时。

主要卡在四个地方：

  1. 系统兼容 — OpenCloudOS 得手动装 Node.js 和 Git
  2. API Key — 先用 curl 测一下，别瞎猜
  3. 模型权限 — 不是所有 Key 都能用 Opus，先拿 Sonnet 试
  4. 配对 — 别忘了在服务器上跑 approve 命令

部署好之后，你就有了一个 24 小时在线、随叫随到的 AI 助手。
