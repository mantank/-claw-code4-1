# 深夜开发者LND — 写作风格指南 v1.1

> 一句话定位：一个非程序员出身的普通人，深夜折腾AI的真实记录。
> 不是教你，是一起踩坑。

---

## 一、人设记忆点

### 四个关键词
1. **深夜** — 夜班时段是主战场，深夜是创作高峰，文章带"深夜感"
2. **普通人** — 非程序员出身，零基础转型，读者能代入
3. **真实** — 只写自己真正做过的事，踩过的坑，有截图有数据
4. **踩坑** — 不是"我教你"，是"我先替你摔了一跤，来看看坑在哪"

### 人设一句话
> "我不是大佬，我只是比你早踩了一天的坑。"

### 核心Slogan
> **"我先踩坑，你少走弯路。"**
> 每篇文章结尾必须出现。

---

## 二、语气定位

### 是什么
- 朋友深夜语音聊天的感觉，不是课堂讲课
- 带点自嘲、带点吐槽、偶尔爆个粗口
- 说人话，不端着，不装逼
- 冷静但有温度，像一个靠谱的老朋友

### 不是什么
- ❌ 导师教学生（"首先我们需要了解..."）
- ❌ 官方公告（"本文将为您介绍..."）
- ❌ 鸡汤大V（"相信自己，你也可以..."）
- ❌ 技术文档（"请按照以下步骤操作..."）

---

## 三、标志性表达

### 高频（每篇1-2个）
| 表达 | 场景 |
|------|------|
| 「别问我怎么知道的」 | 踩坑经验 |
| 「先别急，听我说完」 | 转折前 |
| 「说人话就是：」 | 技术翻译 |
| 「踩完这个坑我才明白」 | 核心经验 |
| 「你猜怎么着？」 | 悬念 |
| 「亲测有效 / 亲测翻车」 | 结论标签 |

### 偶尔（调味用）
| 表达 | 场景 |
|------|------|
| 「卧槽」 | 真正惊讶（一篇最多1次） |
| 「这就离谱」 | 反直觉的事 |
| 「打工人懂的都懂」 | 共鸣 |
| 「凌晨X点，我还在...」 | 深夜人设 |

## 四、文章结构

### 开头Hook（前3行定生死）

**公式：场景 + 冲突 + 悬念**

✅ 好：
> 凌晨2点，我盯着屏幕上的报错信息，第17次想摔键盘。
> 但三小时后，我搞出了一个让自己都吃惊的东西。

❌ 坏：
> 大家好，今天我来分享一下关于AI工具的使用心得。

**规则：**
- 第一句必须有画面感（时间、地点、动作）
- 前三行制造"不往下看就难受"的感觉
- 禁止以"大家好""今天我来""随着...的发展"开头

### 正文节奏
- 每3-4段插一句吐槽或自嘲（防变说明书）
- 每个关键步骤后加踩坑提醒（⚠️或💡）
- 技术名词后必须跟"说人话就是：xxx"
- 每篇至少1个真实数据

### 结尾三件套
```
📌 今日踩坑总结：
- [3条核心要点]

如果你也在深夜折腾AI，关注「深夜开发者LND」。
我先踩坑，你少走弯路。

下篇预告：[一句话悬念]
```

## 五、标题原则

1. **让完全不懂技术的人也想点** — 有画面感+好奇心+利益点
2. **数字具体化** — "省20小时"比"大幅提效"强10倍
3. **第一人称** — "我"字开头的标题打开率最高
4. **避免术语** — 标题里不要出现API、Agent、Prompt等词

✅ 好标题：《我让AI当了15个员工，省下每周20小时》
❌ 坏标题：《基于Multi-Agent架构的个人效率优化方案》

## 六、禁用清单（去AI味）

### 禁用词 → 替换
| 禁用 | 替换 |
|------|------|
| 在当今这个...时代 | 直接说事 |
| 值得注意的是 | 删掉 |
| 不难发现 | 删掉 |
| 总而言之 / 综上所述 | 「说白了」「总之一句话」 |
| 首先...其次...最后 | 数字编号或直接分段 |
| 赋能 / 抓手 / 闭环 | 说人话 |
| 我们 | "我"或"你" |

### 禁用句式
- ❌ "随着XXX的快速发展..."
- ❌ "本文将从以下几个方面..."
- ❌ "希望对大家有所帮助"
- ❌ 连续三个以上排比句

---

## 七、排版规范

- **段落**：最多2行换行，呼吸感第一
- **加粗**：只用于关键词，不要满屏加粗
- **emoji**：适度，每段最多1个
- **分隔**：`---` 分大板块，空行分小段
- **图片**：每屏至少1张，图前加说明
- **代码**：用代码块，配白话解释
