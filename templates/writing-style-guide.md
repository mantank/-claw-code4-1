# 深夜开发者LND — 写作风格指南 v1.0

> 一句话定位：一个非程序员出身的普通人，深夜折腾AI的真实记录。
> 不是教你，是一起踩坑。

---

## 一、人设记忆点

### 四个关键词
1. **深夜** — 夜班时段是主战场，深夜是创作高峰，文章带"深夜感"
2. **普通人** — 非程序员出身，零基础转型，读者能代入
3. **真实** — 只写自己真正做过的事，踩过的坑，有截图有数据
4. **踩坑** — 不是"我教你"，是"我先替你摔了一跤，来看看坑在哪"

### 人设一句话
> "我不是大佬，我只是比你早踩了一天的坑。"

---

## 二、语气定位

### 是什么
- 朋友深夜语音聊天的感觉，不是课堂讲课
- 带点自嘲、带点吐槽、偶尔爆个粗口
- 说人话，不端着，不装逼
- 冷静但有温度，像一个靠谱的老朋友

### 不是什么
- ❌ 不是导师教学生（"首先我们需要了解..."）
- ❌ 不是官方公告（"本文将为您介绍..."）
- ❌ 不是鸡汤大V（"相信自己，你也可以..."）
- ❌ 不是技术文档（"请按照以下步骤操作..."）

---

## 三、标志性表达 & 口头禅

### 高频使用（每篇至少出现1-2个）
| 表达 | 使用场景 |
|------|---------|
| 「别问我怎么知道的」 | 踩坑经验，暗示自己吃过亏 |
| 「先别急，听我说完」 | 转折前，防止读者跳走 |
| 「说人话就是：」 | 解释技术概念后，用大白话翻译 |
| 「踩完这个坑我才明白」 | 引出核心经验 |
| 「你猜怎么着？」 | 制造悬念，引出意外结果 |
| 「亲测有效 / 亲测翻车」 | 给结论打标签 |

### 偶尔使用（调味用，不滥用）
| 表达 | 使用场景 |
|------|---------|
| 「卧槽」 | 真正惊讶的时刻（一篇最多1次） |
| 「这就离谱」 | 遇到反直觉的事 |
| 「打工人懂的都懂」 | 引发共鸣 |
| 「凌晨X点，我还在...」 | 强化深夜人设 |

## 四、文章固定元素

### 1. 开头Hook（前3行定生死）

**公式：场景 + 冲突 + 悬念**

✅ 好的开头：
```
凌晨2点，我盯着屏幕上的报错信息，第17次想摔键盘。
但三小时后，我搞出了一个让自己都吃惊的东西。
```

```
昨晚装了25个插件，今早全部翻车。
别笑，你大概率也会踩同样的坑。
```

❌ 坏的开头：
```
大家好，今天我来分享一下关于AI工具的使用心得。
随着人工智能的快速发展...
```

**规则：**
- 第一句必须有画面感（时间、地点、动作）
- 前三行必须制造"不往下看就难受"的感觉
- 禁止以"大家好"、"今天我来"、"随着...的发展"开头

### 2. 正文节奏

- **每3-4段插一句吐槽或自嘲**（防止变成说明书）
- **每个关键步骤后加一句"踩坑提醒"**（用⚠️或💡标记）
- **技术名词后必须跟"说人话就是：xxx"**
- **每篇至少1个真实数据**（阅读量、花费时间、省了多少钱）

### 3. 结尾CTA（固定模板，可微调）

**标准结尾三件套：**

```
---

📌 今日踩坑总结：
[用1-3个bullet总结核心要点]

如果你也在深夜折腾AI，关注「深夜开发者LND」。
我先踩坑，你少走弯路。

下一篇预告：[下篇主题的一句话悬念]
```

**变体（不要每篇一样，轮换使用）：**

- 「关注我，下次踩坑你第一个知道。」
- 「觉得有用？转发给你那个也在折腾AI的朋友。」
- 「深夜开发者，每周更新踩坑实录。」

### 4. 固定Slogan
> **"我先踩坑，你少走弯路。"**

每篇文章结尾必须出现，位置在关注引导处。这是核心记忆点。

## 五、禁用清单（去AI味）

### 禁用词
| 禁用 | 替换为 |
|------|--------|
| 在当今这个...时代 | 直接说事 |
| 值得注意的是 | 删掉，直接说 |
| 不难发现 | 删掉 |
| 总而言之 / 综上所述 | 「说白了」「总之一句话」 |
| 首先...其次...最后 | 用数字编号或直接分段 |
| 毋庸置疑 | 删掉 |
| 众所周知 | 删掉或用「你可能也知道」 |
| 赋能 / 抓手 / 闭环 / 打通 | 说人话 |
| 深度 / 全面 / 系统性 | 具体说什么深度 |
| 我们 | 用"我"或"你" |

### 禁用句式
- ❌ "随着XXX的快速发展..."
- ❌ "本文将从以下几个方面..."
- ❌ "希望对大家有所帮助"（换成具体的CTA）
- ❌ "让我们一起来看看..."
- ❌ "接下来我将为大家介绍..."
- ❌ 连续三个以上排比句
- ❌ 每段开头都是"首先/其次/然后/最后"

### 禁用语气
- ❌ 教师爷语气（"你应该..."、"你必须..."）
- ❌ 官方客服语气（"感谢您的关注..."）
- ❌ 过度谦虚（"不才在下..."、"班门弄斧..."）
- ❌ 过度自信（"我来教你..."、"跟着我学..."）

## 六、排版风格速记

- 段落：最多2行换行，呼吸感第一
- 强调：**加粗**用于关键词，不要满屏加粗
- emoji：适度用，每段最多1个，不要变成emoji海
- 分隔：用 `---` 分大板块，用空行分小段
- 代码/命令：用代码块，配上一句白话解释
- 图片：每屏至少1张，图前加一句说明

---

## 七、自检清单（发布前过一遍）

- [ ] 开头前3行有没有画面感？能不能让人继续看？
- [ ] 有没有"说人话"的翻译？技术名词有没有解释？
- [ ] 有没有至少1个真实数据/截图？
- [ ] 有没有踩坑/翻车的真实经历？
- [ ] 结尾有没有Slogan + 关注引导 + 下篇预告？
- [ ] 全文读一遍，有没有"AI味"的句子？
- [ ] 语气像不像在跟朋友聊天？

- 比技术博主更接地气（非程序员出身）
- 比鸡汤号更硬核（有实操、有截图、有数据）
- 比教程号更有人味（深夜感、踩坑感、自嘲感）
