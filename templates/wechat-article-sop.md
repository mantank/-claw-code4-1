# 公众号写作SOP v1.0

> 目标：从选题到发布，2小时内完成一篇。
> 迭代自：第1-8篇实战经验总结

---

## 第一步：选题（10分钟）

### 选题来源（按优先级）
1. **群里有人问** → 最高优先，真实需求，转化率最高
2. **自己踩的坑** → 真实经历比编故事强10倍
3. **热点事件+自己的角度** → 蹭热点但要有独立观点
4. **系列教程的下一篇** → 粉丝有预期，完成度高

### 选题检查清单
- [ ] 完全不懂技术的人看标题会想点吗？
- [ ] 我有真实经历/数据支撑吗？
- [ ] 跟竞品文章有差异化吗？
- [ ] 一句话能说清这篇讲什么吗？

---

## 第二步：标题（10分钟）

### 标题铁律
- 让完全不懂技术的人也想点进来
- 有画面感 + 好奇心 + 利益点
- 第二篇技术标题只有2阅读 vs 第一篇通俗标题63阅读——血的教训

### 标题公式
- **反差型：** 「别瞎装！xxx安全避坑」「我让AI接管了xxx，它比我自己做得还好」
- **数字型：** 「25个技能全公开」「3分钟搞定xxx」
- **身份型：** 「一个非程序员的xxx实录」
- **悬念型：** 「昨晚，我给自己招了一个AI员工」

### 出3个标题，选最"想点"的那个

---

## 第三步：大纲（10分钟）

### 结构模板
```
钩子（为什么写这篇）
↓
核心概念（一句话讲清楚）
↓
实操步骤（手把手）
↓
踩坑经验（真实教训）
↓
结尾引导（关注+预告下一篇）
```

### 大纲检查
- [ ] 每个板块能用一句话概括吗？
- [ ] 有没有真实经历/截图支撑？
- [ ] 读者看完能立刻动手操作吗？

---

## 第四步：正文（60分钟）

### 排版铁律
- 每段不超过2行（超过就没人读）
- 一屏 = 2/3文字 + 1张图
- 字不能太小，板块要清晰
- 不要超过手机一屏不切换

### 写作原则
- 说人话，不说术语（必须用术语时加一句解释）
- 用"我"不用"我们"，用"你"不用"用户"
- 真实 > 完美（说踩了什么坑比说多厉害更有信任感）
- 每个步骤都要可验证、可复制

### 配图位置标注
写正文时用 📸【截图X】标注需要配图的位置，写完统一截图。

---

## 第五步：配图（20分钟）

### 封面图
- 风格：卡通插画（匹配老大的数字形象）
- 工具：Gemini 3 Pro Image（evolink）/ ChatGPT
- 尺寸：公众号头图比例

### 文内图
- 对话截图：Telegram聊天记录直接截
- 操作截图：浏览器/终端实际操作画面
- 数据图：如有数据对比，做简单图表
- 每张图都要有上下文（图前一句话说明这是什么）

---

## 第六步：排版发布（10分钟）

### 工具
- Markdown Nice (editor.mdnice.com) → 复制到公众号编辑器

### 发布前检查
- [ ] 标题够吸引人吗？（再看一遍）
- [ ] 所有截图都插入了吗？
- [ ] 读一遍，有没有读不下去的地方？
- [ ] 结尾有引导关注吗？
- [ ] 合集标签设置了吗？

### 发布后
- [ ] 分享到至少5个渠道（群聊/朋友圈/社群/知识星球/...）
- [ ] 记录到Notion公众号文章库
- [ ] 24小时后看数据（阅读/分享/关注）

---

## 时间分配总结

| 步骤 | 时间 | 备注 |
|------|------|------|
| 选题 | 10min | 平时积累选题池，写的时候直接选 |
| 标题 | 10min | 出3个选1个 |
| 大纲 | 10min | 让零零壹先出大纲再改 |
| 正文 | 60min | 核心时间，零零壹写初稿→老大改 |
| 配图 | 20min | 封面AI生成，文内截图 |
| 排版发布 | 10min | Markdown Nice一键排版 |
| **合计** | **2h** | |

---

## 协作模式

1. 老大提选题/灵感 → 零零壹出大纲
2. 老大确认/调整大纲 → 零零壹写初稿
3. 老大审稿+提修改意见 → 零零壹改V2
4. 老大截图+排版+发布
5. 零零壹同步到Notion/飞书+记录数据
