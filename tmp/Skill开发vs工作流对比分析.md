# Skill 开发 vs n8n/Coze 工作流：对比分析

**分析人：** 零零壹（001）
**日期：** 2026-03-07

---

## 核心结论

**n8n/Coze 是"你告诉 AI 每一步怎么走"，Skill 是"你告诉 AI 它会什么，它自己决定怎么走"。**

去年卖 n8n 工作流模板赚钱的那批人，今年的机会就是卖 Skill。门槛确实降低了，但本质变了。

---

## 三者对比

### n8n（去年主流）
- **核心思路：** 流程编排，A→B→C→D，每一步你设计好
- **谁做决策：** 你提前设计好每一步
- **灵活性：** 低，流程固定，需求变了要改流程
- **技术门槛：** 中等（要会拖拽+调API+调试连线）
- **部署方式：** 云端或自建服务器
- **数据隐私：** 自建可控，云端不可控
- **商业模式：** 卖模板/卖搭建服务

### Coze/Dify（过渡期产品）
- **核心思路：** 可视化搭 Bot
- **谁做决策：** 你设计好流程 + 少量 AI 判断
- **灵活性：** 中等
- **技术门槛：** 低（可视化操作）
- **部署方式：** 云端（字节托管）
- **数据隐私：** 数据在字节服务器
- **商业模式：** 卖 Bot/卖搭建服务

### ClawHub Skill（当前机会）
- **核心思路：** 能力封装，给 Agent 加技能
- **谁做决策：** Agent 自己判断什么时候用什么 Skill
- **灵活性：** 高，Agent 根据上下文自主调用
- **技术门槛：** 最低（写 prompt + 定义工具就行）
- **部署方式：** 本地私有化（OpenClaw 内置）
- **数据隐私：** 完全本地，数据不出你的机器
- **商业模式：** 卖 Skill/卖行业方案

---

## 核心区别：确定性 vs 智能性

### n8n 工作流（去年的玩法）

```
触发器 → 抓取数据 → 调 GPT 处理 → 发到飞书
```

每一步都是提前设计好的，跑得稳但死板。客户需求变一点，就得改流程。

### Skill（今年的玩法）

```
你只定义：这个 Skill 能"读飞书文档"、能"生成报告"
Agent 自己判断：
  用户说"帮我总结今天的会议纪要"→ 自动调飞书读文档 → 自动生成报告
```

不用设计流程，Agent 自己组合能力。

### 打个比方

- n8n 像是给员工写了一本操作手册，每一步都规定死
- Skill 像是培训员工一项技能，他自己判断什么时候用

---

## 去年的 n8n 市场发生了什么

去年卖 n8n 工作流的人确实赚了一波：
- 一套"小红书自动发帖工作流"卖 299-999
- "外贸客户跟进自动化"卖 1999-4999
- 搭建服务 3000-10000/单

**但现在这个市场在萎缩，三个原因：**

1. **Coze/Dify 把门槛降到零** — 会打字就能搭，n8n 的"中等门槛"变成了劣势
2. **Agent 框架（OpenClaw）把上限拉高** — 工作流只能跑固定流程，Agent 能处理不确定的场景
3. **客户变聪明了** — 去年花 5000 买一套工作流，今年发现改一下需求就要再花钱，不划算

---

## Skill 开发的机会在哪

去年卖 n8n 模板 = 今年卖 Skill，但有三个关键升级：

### 1. 开发更简单
- n8n 要懂 API、要拖节点、要调试连线
- Skill 本质就是一个 SKILL.md 文件 + 一段 prompt + 几个工具定义
- **你现在天天在做的事（配 Agent、写 prompt），就是在做 Skill 开发**

### 2. 分发渠道更好
- n8n 模板卖在哪？自己开淘宝店、发小红书、挂闲鱼，获客全靠自己
- Skill 上架 ClawHub，月均 24 万独立访客，自带流量

### 3. 壁垒更高
- n8n 模板买了就能抄，没有壁垒
- Skill 里封装的是行业 know-how（比如你懂物业流程），这个抄不走

## 实际案例对比

### 场景：帮物业公司自动处理业主报修

**n8n 做法（去年）：**
```
微信收到消息 → 关键词匹配"报修" → 调 GPT 提取地址+问题 → 写入飞书表格 → 通知维修工
```
固定流程，业主说了一句含糊的话就识别不了，需要手动加分支。

**Skill 做法（今年）：**
```
Skill 定义：能读微信消息、能写飞书表格、能发通知
Agent 自己判断：
- 业主说"水管又漏了" → 识别为报修 → 自动分派
- 业主说"上次修的不行" → 识别为投诉 → 升级处理
- 业主说"物业费怎么交" → 识别为咨询 → 自动回复
```
不用提前设计所有分支，Agent 自己理解意图。

## 总结判断

**Skill 替代了 n8n 吗？**
没完全替代，但抢了 80% 的场景。n8n 还适合确定性高、不需要 AI 判断的纯自动化（比如数据同步）。

**Coze 还有价值吗？**
有，但被 OpenClaw 压制——Coze 数据在云端，OpenClaw 本地私有化。

**去年卖工作流的人今年该干嘛？**
转做 Skill 开发，把行业经验封装进去。

**对旭来说意味着什么？**
不需要学 n8n，直接做 Skill 就行。现在给自己搭的 Agent 流程，稍微打包一下就是一个可卖的 Skill。

---

**一句话总结：n8n 是上一代的"卖铲子"，Skill 是这一代的"卖铲子"。铲子升级了，但挖金矿的逻辑没变。**
