# 小红书自动发布调研（2026-03-23）

## 现有工具盘点

### 1. xiaohongshu-mcp
- **性质**：Python客户端 + 本地MCP服务器
- **能力**：搜索/发布/查详情/评论
- **关键限制**：
  - 需要在**本地Mac**运行，服务器（腾讯云43.159.48.54）被小红书封禁
  - 需要扫码登录（cookie/session有效期问题）
  - 每次发布需要图片路径
- **状态**：Mac本地可用，服务器不可用

### 2. xiaohongshu-ops-skill
- **性质**：运营SOP流程skill
- **能力**：选题/竞品分析/内容模板/发布流程规范
- **关键限制**：
  - 明确要求"到发布页停手"，不自动点击发布
  - 强调人工确认后才发布
- **状态**：已安装可用，但不支持全自动

### 3. baoyu-xhs-images
- **性质**：图片生成skill（白板风/卡通风格）
- **能力**：生成小红书风格的系列图片（10种风格×8种布局）
- **状态**：已安装，用于配图

## 自动化可行性分析

### ✅ 可自动化部分
- 选题生成（用LLM+搜索）
- 内容文案撰写（002 already does）
- 图片生成（Nano Banana/Gemini）
- 草稿准备好，等待人工确认发布

### ❌ 无法全自动的部分
- **发布动作**：小红书有强反自动化检测，自动化发布容易被封号
- **扫码登录**：cookie有效期短，需定期重新认证
- **图片上传**：需要真实浏览器环境

## 建议方案：半自动流水线

```
002生成选题+文案+配图方案
    ↓
001审核/润色
    ↓
图片生成（baoyu-xhs-images）
    ↓
人工确认（旭在手机上操作发布）
```

**不是全自动发布，而是把人从"写文案"中解放出来。**

## 风险提示
- 小红书对账号批量操作敏感度高
- 半自动模式（人确认+发布）更安全，不容易封号
- 反自动化检测强，全自动发布风险高
