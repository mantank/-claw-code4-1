# 公众号冷启动研究笔记
> 2026-03-01 001主动研究

## 核心结论（适用于深夜开发者LND）

### 我们的优势
- 有真实做事过程（搭AI团队），天然是最好的内容素材
- 傅盛模式完全可以复制：做事→故事→内容
- 不是在"写文章"，是在"记录探索"，降低创作门槛

### 冷启动关键认知
1. **前10篇决定账号生死** — 每篇都要认真，不能凑数
2. **精准1000粉 > 泛流量10000粉** — 找到真正关心AI+普通人场景的读者
3. **完成比完美重要** — 先跑通发布节奏，再优化质量

### 从30→200粉的最快路径（我们场景）

**方法1：蹭热点（最快）**
- 每次有大模型发布/AI大事件，第一时间出"普通人视角"解读
- 今天就有：Claude被封禁反而冲上App Store第2——这是天然热点
- 模板："xxx发生了，对我们普通人意味着什么？"

**方法2：做事→故事（最持续）**
- 把自己搭AI团队的过程写出来，真实案例比教程更有传播力
- 类似傅盛龙虾日记，但我们的视角是"普通非程序员也能干"
- 每次踩坑都是一篇文章

**方法3：互动导流（低成本）**
- 在知识星球（生财有术）发精华帖，引流到公众号
- 在大V文章评论区留有价值的评论，主页引流
- 不要刷互粉，要找真实目标用户

### 目前缺失的
- ❌ 没有固定的"钩子"：关注后能得到什么？（现在关注后什么也没有）
- ❌ 没有引流入口：只靠搜索，没有主动导流渠道
- ❌ 发布频率不稳定：要么不发，要么扎堆发

### 建议行动
1. 设计一个"关注送X"钩子（一份AI工具清单？一个提示词模板？）
2. 把知识星球作为主要引流渠道，每次发帖带公众号
3. 固定每周二+周五发布，建立读者预期

## 小红书自动发布调研

### 可用方案

**方案1：xiaohongshu-mcp（推荐）**
- GitHub: xpzouying/xiaohongshu-mcp
- 技术：Playwright + MCP Server
- 能力：登录、发图文、发视频、搜索、评论
- 我们已安装xiaohongshu-mcp skill（需验证是否是这个）
- 接入方式：`claude mcp add --transport http xiaohongshu-mcp http://localhost:18060/mcp`

**方案2：n8n + MCP工作流**
- 可视化工作流：选题→创作→发布→监测全自动
- 更重，但更灵活
- 当前阶段不需要，先用MCP直接接

**方案3：Auto-Redbook-Skills（GitHub开源）**
- 集成大模型+图像生成，自动撰写笔记+生成图片+发布
- 全流程自动化，但需要部署

### 结论
- 小红书自动发布技术上可行，xiaohongshu-mcp是最轻量的方案
- 我们已有xiaohongshu-mcp skill，需要验证能否在服务器上跑起来（需要浏览器环境）
- **风险**：小红书反自动化检测较强，建议半自动（AI写内容，人工发布）而非全自动
