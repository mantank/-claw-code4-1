## 二、写作风格特征

### 1. 开头模式

**现状（差异明显）：**
- ✅ 最好的开头：「凌晨两点，Claude Code 4.6 的发布了。说实话，有点兴奋，又有点慌。」——用时间+情绪制造画面感，读者立刻代入
- ⚠️ 普通的开头：「大家好 我是」+ 直接介绍内容——太平，开头没有钩子

**统一方向：**
- 开头必须有「场景」或「情绪」，不用介绍自己，直接带感觉
- 第二句必须有冲击力：数据、反常识、或强烈情绪对比
- ✅ 好的示例：「凌晨两点，我让一个AI帮我装了另一个AI。」
- ❌ 避免：「今天给大家分享一篇...」「本篇内容主要是...」

### 2. 标题风格

**已有的好模式：**
- **冲突对比型**：「Claude Code 4.6 很强，但真正拉开差距的，不是模型」（最高阅读）
- **警示+数字**：「别乱装！三步排雷法+25个清单+3个踩坑案例」
- **感叹号强调**：「VS Code神器搭配！...告别Cursor！」

**升级方向：**
- 加入「反常识」感：读者看了想"真的假的？"然后点进来
- ✅「我把AI管理工作甩给了另一个AI，结果比我整理得好」
- ❌「OpenClaw飞书功能详细教程」（太平，没张力）
- 避免"如何""为什么"开头

### 3. 人称和语气

- **第一人称贯穿**：「我」「我的」「我发现」
- **真实感强**：不装专家，直说自己踩坑、懵逼、然后搞通的过程
- **口语化短句**：「你会说这个早就可以了呀！对，但是...」（对话感好）
- **有态度**：「那我为什么还要用一个二手的呢，对吧！」——敢下判断

### 4. 内容呈现

- **步骤类**：有序列表，编号清晰（01/02/03或1、2、3）
- **痛点引入**：先说读者的困境，再给解决方案（文章3经典示例）
- **类比接地气**：「就像手机装App——不装App的iPhone只能打电话」
- **before vs after**：「以前...现在...」结构出现较多，是强项
- **短句节奏**：最好的段落每行不超过2行，有呼吸感

### 5. 文章结构

**教程类（最多）**：
```
痛点开头（你是否也有这个问题？）
→ 先给结论/效果（结果前置）
→ 为什么会这样（背景）
→ 步骤一二三（带截图说明）
→ 我的真实体验
→ 总结/行动引导
```

**深度类（潜力最大）**：
```
场景/情绪开头（凌晨两点...）
→ 情绪共鸣（我的困惑）
→ 一个核心问题/洞察
→ 分析+观点（敢下判断）
→ 给读者的启示
→ 弹射式结尾（强判断收束）
```

## 三、当前最强的文章

**「Claude Code 4.6 很强，但真正拉开差距的，不是模型」**

为什么最强：
- 开头有画面感+情绪共鸣
- 语气自然，不装
- 核心观点「不是技术问题，是管理思维问题」有原创性
- 短句密度高，节奏好

→ **这篇是风格基准。后续所有文章向这篇看齐。**

---

## 四、需要升级的三个地方

### ① 开头命中率不稳定
有些文章开头太平，失去读者。
**解决**：每篇必须用「场景+情绪」开头，不用介绍自己。

### ② 金句密度不够
每500字需要有一句「可截图转发」的狠话/短句。
**示例**：
- 「普通人拿到一把屠龙刀，却不知道龙在哪。」（已有，很好）
- 「AI越来越强，会用AI的人越来越值钱。」（可加）

### ③ 结尾缺乏「弹射」感
结尾应该用强判断或反问收束，不要平和总结。
- ✅ 弹射式：「工具就摆在那里。用不用，是你的事。」
- ❌ 软绵绵：「希望这篇文章对大家有帮助，我们下期再见。」

## 六、活人感原则（卡兹克方法论）

> AI时代最贵的奢侈品是「活人感」。AI能写分析，但写不了你凌晨折腾 OpenClaw 时的那种真实感受。

**什么是活人感：**
- 不是技巧，是你的情感、偏见、成长、笨拙在文字里的自然流露
- 你的不完美、你的主观判断，是 AI 无法复现的护城河
- 可以不完美，但必须是真实的「你」

**具体执行：**
- 写任何工具测评，从「我遇到了什么问题」开始，而不是「这个工具有什么功能」
- 写观点文，敢说「我认为/我觉得」，不装客观
- 写经历，带上情绪起伏：好奇→踩坑→解决→感悟

## 七、HKR 发布前自检清单（卡兹克框架）

每篇发布前对照以下3问，至少通过2个：

```
H（Hook/Happiness）：
  □ 开头5秒能勾住人吗？
  □ 标题让人想"卧槽，还有这种事？"
  □ 有趣，让人想继续读下去

K（Knowledge）：
  □ 读完能学到新东西吗？
  □ 有没有读者90%不知道的信息/角度
  □ "原来是这样，学到了"

R（Resonance）：
  □ 有没有让读者说"对对对，我也这样"的瞬间
  □ 有没有情绪共鸣点（焦虑/好奇/兴奋）
  □ 读完想截图发给朋友吗

结论：
✅ 3个全过 → 有爆款潜力
✅ 2个通过 → 合格，可发
⚠️ 只过1个 → 需要修改
❌ 全不过  → 不要发
```

## 八、读者画像升级版

不只是「非技术人员」，更精准的是：

**你的读者是：聪明、选择很多、很忙的人**

- **聪明**：不要讲废话，不要把读者当傻子，复杂的讲清楚而不是简单的反复说
- **选择很多**：他关注了无数博主，选择读你是放弃刷视频、看剧的机会成本，你的内容必须值得
- **很忙**：结构清晰，重点加粗，一目十行也能抓到核心

**写作操作意义：**
- 每段不超过2行，节奏快
- 关键信息加粗，不埋在段落里
- 结构一眼可见（标题/编号/空白行）

## 九、爆款公式

```
开头：场景+情绪（制造画面感，第一句让人停下来）
中间：before vs after + 具体数据/案例
金句：每500字一句短狠句
结尾：强判断/反问（让读者想截图发朋友圈）
篇幅：1500-2500字（宁短勿水）
```
