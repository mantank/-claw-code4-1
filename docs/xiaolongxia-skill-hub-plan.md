# xiaolongxia.app 技能应用页面方案

> 制定时间：2026-03-13
> 定位：技能的"大众点评"，不是技能的"应用商店"
> 核心差异：腾讯告诉你"有这个菜"，我们告诉你"怎么做、好不好吃、踩过什么坑"

---

## 一、定位与差异化

### 竞品现状

| 平台 | 做了什么 | 没做什么 |
|------|---------|---------|
| skillhub.tencent.com | 1.3万技能目录、国内CDN、安全审计、TOP50榜 | 无案例、无教程、无踩坑、无组合推荐 |
| clawhub.ai | 2万技能注册、版本管理、安全扫描、评分 | 纯英文、无使用案例、无截图演示 |
| claw123.ai | 5484精选技能、恶意过滤、Bot友好API | 纯导航站、无内容、点击跳外站 |

### 我们的定位

**「技能实战间」—— 不列技能参数，讲技能怎么用、用了什么效果、踩了什么坑**

用户来这里不是看"这个技能叫什么"，而是看：
- 这个技能能帮我解决什么问题？
- 别人是怎么配的？Prompt能不能直接抄？
- 有什么坑要注意？
- 跟哪些技能搭配效果更好？

## 二、页面架构

### 2.1 整体结构

```
xiaolongxia.app
```

## 三、数据架构

### 3.1 数据来源（零人工维护）

```
案例数据（已有79个）
    ↓ 自动提取 tools 字段
技能引用关系（自动聚合）
    ↓ 统计引用次数
技能页面数据（自动生成）
    + 手工补充：踩坑提醒、常见搭配、一句话描述
```

**关键：技能列表不是手动维护的，是从案例的 `tools` 字段自动聚合的。**
- 案例越多，技能页面内容越丰富
- 新案例发布 → 技能页面自动更新引用数
- 零维护成本

### 3.2 数据Schema

```typescript
// 技能数据（手动维护的部分，存在 /content/skills/ 目录）
interface SkillMeta {
  slug: string;              // feishu-bitable
  name: string;              // 飞书多维表格
  nameEn: string;            // feishu-bitable
  oneLiner: string;          // 一句话描述
  category: Category;        // 场景分类
  difficulty: 1 | 2 | 3;     // 难度星级
  timeSaved?: string;        // 效果量化
  clawhubUrl?: string;       // ClawHub安装链接
  tencentUrl?: string;       // 腾讯镜像链接
  docUrl?: string;           // 官方文档链接
  combos: Combo[];           // 常见搭配
  pitfalls: string[];        // 踩坑提醒
}

interface Combo {
  partner: string;           // 搭配的技能slug
  effect: string;            // 搭配效果一句话
}

// 自动聚合的部分（构建时从案例中提取）
interface SkillAggregated {
  caseCount: number;         // 引用此技能的案例数
  cases: CaseRef[];          // 关联案例列表
  categories: string[];      // 涉及的场景分类
}

// 组合套餐（手动维护，存在 /content/combos/ 目录）
interface SkillCombo {
  slug: string;              // wechat-auto-publish
  name: string;              // 公众号全自动写稿套餐
  skills: string[];          // 包含的技能slug列表
  description: string;       // 能干什么
  effect: string;            // 效果量化
  difficulty: 1 | 2 | 3;
  audience: string;          // 适合谁
  caseSlug?: string;         // 关联的完整案例
}

type Category = 
  | '社交与内容'
  | '办公效率'
  | '生活助手'
  | '开发工具'
  | '数据与监控'
  | '入门必装';
```

## 六、与现有页面的关系

```
用户路径A（从案例出发）：
  场景案例页 → 看到案例用了哪些技能 → 点技能 → 看这个技能还能干什么

用户路径B（从技能出发）：
  技能实战间 → 浏览技能 → 看实战案例 → 复制Prompt去用

用户路径C（从组合出发）：
  组合套餐页 → 看到一整套方案 → 点进去看完整配置 → 一次装好几个技能

三条路径交叉引用，互相导流，核心数据是同一份案例库。
```

### 与外部平台的关系

```
xiaolongxia.app（我们）          腾讯SkillHub / ClawHub（他们）
─────────────────────           ──────────────────────────
讲"怎么用"                       讲"是什么"
讲"踩了什么坑"                   讲"参数规格"
讲"搭配什么更好"                  讲"下载量排名"
提供案例+Prompt                  提供安装命令+版本

互补关系：安装链接直接指向他们，我们不重复做安装功能
```
