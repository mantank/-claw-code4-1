## 一、这个网站是什么

**一句话定义：** 国内用户获取 OpenClaw 玩法、教程、技能和应用案例的中文第一入口。

**更详细地说：** 很多国内用户装好了 OpenClaw，但不知道怎么用、用来干什么。GitHub 上不去、Discord 看不了、官方文档全英文。这个网站就是帮他们解决"装好了，然后呢？"这个问题。

**不是什么：**
- 不是 OpenClaw 官方网站（我们是社区站）
- 不是技术文档站（不教你写代码）
- 不是纯工具导航（WaytoAGI 已经做了）

**是什么：**
- OpenClaw 的中文使用指南 + 技能商店 + 真实案例库 + 用户社区
- 面向**不会写代码的普通用户**
- 所有内容的标准：看完就能用，不需要懂技术

## 二、给谁用

### 用户画像

| 用户类型 | 他是谁 | 他的痛点 | 他来干什么 |
|----------|--------|----------|-----------|
| **小白用户** | 刚听说 OpenClaw，好奇想试 | 不知道 OpenClaw 能干什么、怎么装 | 看教程、看案例、判断值不值得折腾 |
| **已装用户** | 装好了但只会基本对话 | 不知道有哪些技能、怎么让 AI 干更多事 | 找技能、看玩法、一键安装 |
| **进阶用户** | 想用 OpenClaw 做业务（自媒体/电商/办公） | 找不到适合自己行业的方案 | 看行业套装、看别人怎么用的、交流经验 |

### 核心场景（用户来了干什么）

1. **"我想让 AI 帮我发小红书"** → 搜索/浏览技能商店 → 找到小红书技能 → 看使用教程 → 复制安装命令
2. **"OpenClaw 到底能干嘛"** → 看首页精选案例 → 被某个案例打动 → 点进去看详细教程
3. **"我是做跨境电商的，有没有现成方案"** → 浏览行业套装 → 找到跨境电商套装 → 看包含哪些技能和配置
4. **"我搞了个好玩的用法想分享"** → 提交案例（v2 功能）

## 四、核心板块详细设计

### 4.1 技能商店（最重要的板块）

**定位：** 让用户像逛应用商店一样找 OpenClaw 技能，看完就能装。

**技能卡片包含：**

| 字段 | 说明 | 示例 |
|------|------|------|
| 技能名称 | 中文名（英文原名） | 小红书自动发布（xhs-publisher） |
| 一句话介绍 | 这个技能干什么 | 让 AI 帮你写笔记、配图、一键发布到小红书 |
| 分类标签 | 所属场景 | 内容创作、社交媒体 |
| 难度等级 | ⭐~⭐⭐⭐ | ⭐ 零基础可用 |
| 安装命令 | 一键复制 | `clawhub install xhs-publisher` |
| 使用教程 | 图文步骤 | 装好后怎么用，分 3 步说清楚 |
| 效果展示 | 截图/动图 | 实际运行效果 |
| 来源 | clawhub / GitHub / 社区自制 | clawhub.com |

**分类体系：**

| 分类 | 包含内容 |
|------|---------|
| 📱 内容创作 | 小红书、公众号、Twitter/X、视频脚本、配图生成 |
| 💼 办公提效 | 邮件管理、日程安排、文档整理、会议纪要 |
| 🛒 电商运营 | 跨境电商、闲鱼、商品文案、数据监控 |
| 🔧 开发工具 | 代码审查、API 调试、部署自动化 |
| 🏠 生活助手 | 天气、提醒、智能家居、记账 |
| 📊 数据分析 | 爬虫、报表、趋势分析 |
| 🎨 创意设计 | 图片生成、PPT、品牌设计 |

### 4.2 教程中心

**设计原则：** 所有教程假设读者**零技术背景**，每一步都有截图，术语都有中文解释。

**教程分级：**

| 级别 | 目标 | 内容 |
|------|------|------|
| 🟢 入门 | 从零到能用 | OpenClaw 是什么 → 怎么安装 → 第一次对话 → 连接手机 |
| 🟡 进阶 | 从能用到好用 | 安装技能 → 定时任务 → 多 Agent → 连接飞书/微信 |
| 🔴 实战 | 从好用到生产力 | 完整场景教程（如：用 OpenClaw 自动运营小红书账号） |

### 4.3 应用案例

**内容来源：**
- 003 从 GitHub/Discord/X 采集的国外案例（翻译+本地化）
- 我们自己的真实使用经验（深夜开发者的龙虾故事）
- 用户投稿（v2 功能）

**案例卡片包含：**
- 标题（吸引人的一句话）
- 场景标签（自媒体/电商/办公/...）
- 概要（3 句话说清楚干了什么、效果怎么样）
- 详情（完整故事 + 配置方法 + 效果数据）

### 4.4 行业套装

**定位：** 不想一个个找技能的用户，直接给他一整套方案。

**套装示例：**

| 套装名 | 目标用户 | 包含内容 |
|--------|---------|---------|
| 🎯 自媒体全家桶 | 个人 IP / 自媒体人 | 公众号发布 + 小红书 + 选题采集 + 配图生成 + 定时发布 |
| 🛒 电商运营包 | 电商卖家 | 商品文案 + 竞品监控 + 客服自动回复 + 数据报表 |
| 💼 办公效率包 | 职场人 | 邮件管理 + 日程提醒 + 会议纪要 + 文档整理 |
| 🎓 学习助手包 | 学生 / 终身学习者 | 论文阅读 + 笔记整理 + 知识卡片 + 每日学习计划 |

每个套装页面包含：
1. 这套方案能帮你做什么（效果描述）
2. 包含哪些技能（列表 + 各自作用）
3. 怎么一步步装好（保姆级教程）
4. 真实效果展示（截图/数据）

### 4.5 每日精选

**更新频率：** 每天 1-3 篇

**内容来源：**
- 003 自动从国外采集 + 翻译
- 001 编辑润色 + 本地化
- 旭审核把关

**内容类型：**
- 🔥 新技能推荐（clawhub 上新了好东西）
- 📖 玩法教程（国外大神的骚操作翻译版）
- 💡 使用心得（我们自己踩过的坑）
- 📊 生态动态（OpenClaw 更新了什么、社区出了什么新东西）

## 五、先做什么（MVP 范围）

### v1（第一版，先上线跑起来）

**只做 3 个页面：**

| 页面 | 理由 |
|------|------|
| ✅ 首页 | 门面，必须有 |
| ✅ 技能商店 | 最硬的差异化，用户来了有东西可用 |

| 页面 | 为什么后做 |

| ⏸ 搜索功能 | 内容少的时候手动分类就够了 |
| ⏸ 评论/评分 | 需要用户系统 |

### v1 的种子内容（上线时至少需要）

## 六、视觉风格

### 整体调性
- **简洁、现代、轻量** — 不要花哨，不要酷炫动画
- **参考：** WaytoAGI 的紧凑卡片式布局 + sanwan.ai 的故事化展示
- **配色：** 主色用 OpenClaw 品牌色（蓝绿色系），辅以中性灰白
- **字体：** 中文用系统默认（苹方/思源黑体），英文用 Inter
- **图标：** 用 emoji 代替复杂图标（降低设计成本，增加亲和力）

### 关键原则
1. **移动端优先** — 国内用户大多数用手机浏览
2. **加载要快** — 静态站优先，不要重前端框架
3. **文字要大** — 目标用户不是程序员，阅读体验优先
4. **一键复制** — 安装命令、配置代码都要有复制按钮

## 七、技术方案

### 推荐方案

| 项目 | 选择 | 理由 |
|------|------|------|
| **框架** | Astro 或 Next.js | Astro 更轻（静态为主），Next.js 更灵活（后续加动态功能） |
| **样式** | Tailwind CSS | 快速开发，响应式内置 |
| **内容管理** | Markdown 文件 | 初期最简单，003 直接输出 MD 文件就能发布 |
| **部署** | Vercel 或 Cloudflare Pages | 免费、国内访问速度可接受、自动部署 |
| **域名** | xiaolongxia.app | ✅ 已注册（Cloudflare） |
| **CDN/加速** | Cloudflare | 免费 CDN，国内访问友好 |

### 内容更新流程（v1）

```
003 采集英文内容
    ↓
001 翻译 + 本地化 + 写成 Markdown
    ↓
Git 提交到仓库
    ↓
Vercel/Cloudflare 自动部署
    ↓
网站更新
```

### 后续扩展（v2+）
- 加数据库（Supabase / PlanetScale）存用户数据
- 加后台 CMS（方便旭直接管理内容）
- 加 API（供 003 自动提交内容）
