## 一、产品定位

### 一句话

**「养一只你的AI龙虾」—— OpenClaw中文实战站 + 游戏化Agent管理面板**

### 两面一体

| 公开面（内容站） | 私有面（龙虾牧场） |
|----------------|-----------------|
| 所有人可访问 | 接入自己的OpenClaw后可用 |
| 龙虾秘籍/副本/新手村/套装 | 实时看自己的龙虾在干什么 |
| 学习怎么养龙虾 | 真的在养龙虾 |
| 内容驱动SEO获客 | 工具驱动用户留存 |

两面合一后的体验：
- 看完秘籍 → 一键给自己的龙虾装上
- 看完副本攻略 → 直接让龙虾去刷
- 龙虾刷副本时遇到Boss → 跳转踩坑攻略

### 竞品空白

| 现有竞品 | 做了什么 | 没做什么 |
|---------|---------|---------|
| skillhub.tencent.com | 技能目录+国内CDN | 无案例、无管理、无游戏感 |
| clawhub.ai | 技能注册+安全扫描 | 纯英文、无使用指导 |
| claw123.ai | 精选技能导航 | 纯导航、无内容 |
| awesome-usecases | 真实案例+Prompt | 纯GitHub、无设计、无管理 |
| OpenClaw Dashboard | 官方管理面板 | 技术风、无游戏感、无内容 |

**我们的唯一性：中文 + 像素RPG游戏感 + 真实案例 + Agent管理面板 = 没有人做过**

## 二、视觉风格规范

### 2.1 风格定义

**星露谷物语（Stardew Valley）× 龙虾养成 × AI工具站**

关键词：
- 像素风（16-bit复古游戏感）
- 温暖治愈（不是冷硬科技感，是农场经营的温馨感）
- 成就驱动（每个操作都有"叮！"的满足感）
- 简洁实用（像素风是外衣，内容实用性是核心）

### 2.2 色彩方案

```
主色调：
  草地绿   #4CAF50 / #66BB6A   主背景、按钮、正面状态
  泥土棕   #8D6E63 / #A1887F   卡片边框、面板底色
  天空蓝   #64B5F6 / #90CAF9   顶栏、链接、蓝条(MP)

强调色：
  龙虾红   #EF5350 / #E53935   龙虾角色、血条(HP)、重要标记
  金币黄   #FFD54F / #FFC107   星级、成就、高亮、金甲装备
  经验紫   #AB47BC / #9C27B0   经验值、进阶内容

中性色：
  面板米色  #FFF8E1 / #FFFDE7  内容区背景
  像素黑   #212121             文字、边框
  阴影灰   #9E9E9E             次要文字

禁止使用：
  ✗ 渐变色（破坏像素感）
  ✗ 高饱和度霓虹色
  ✗ 纯白大面积背景
  ✗ 毛玻璃/模糊效果
```

### 2.3 字体规范

```
标题/导航/按钮/标签：像素字体
  首选：Zpix（中文像素字体，开源免费）
  备选：丁卯点阵体
  英文：Press Start 2P（Google Fonts）

正文/教程内容/代码块：现代字体（保证长文可读性）
  首选：思源黑体 / Noto Sans SC
  
规则：
  标题、导航、按钮、Badge → 像素字体
  正文、教程、代码 → 现代字体
  两种字体混搭 = 像素风外衣 + 实用内核
```

### 4.5 数据架构

```
    ↓ 构建时自动提取
技能引用关系（自动统计）
    ↓ 合并
技能元数据（手工维护：一句话描述+踩坑+搭配）
    ↓ 生成
```


### 8.2 OpenClaw Gateway API（已有，不用额外开发）

```
GET  /api/sessions         → 所有session状态
GET  /api/sessions/:id     → 单个session详情
POST /api/sessions/:id/send → 发消息给Agent
GET  /api/agents           → Agent列表
WS   /api/events           → 实时事件流

Gateway端口：
  001: 18789
  002: 18790
  003: 18791
  004: 18792
```

### 8.3 牧场接入方式

```
方式A（本地用）：
  用户在自己机器上打开 localhost:端口
  直连本地Gateway，无需暴露公网

方式B（远程接入）：
  用户填入Gateway公网地址（或用Cloudflare Tunnel）
  xiaolongxia.app前端直接调用用户的Gateway API
  需要考虑CORS和认证

方式C（SaaS模式，远期）：
  用户把Gateway连到我们的中转服务
  统一管理面板
```
