5个板块，15-20条行业动态，每条带来源链接和分析，全部是过去24小时的真实信息。不是AI编的，是AI搜的。

## 需要什么

一台装好OpenClaw的服务器（或电脑），仅此而已。

OpenClaw是一个AI Agent框架，能让AI自动执行任务，支持定时运行和消息推送。如果你还没装，先参考官方文档把它跑起来。

下面是完整的搭建步骤。

## 第一步：想清楚你的日报要什么

在动手之前，先定两件事：

**定板块。** 你每天最需要了解哪几类信息？

我的AI行业日报分了5个板块：

- 模型速报 — 大模型发布、更新、评测
- AI工具和应用 — 新工具、重大功能更新
- 博主精读 — 行业KOL的深度文章和观点
- GitHub精选 — 开源项目和技术趋势
- 今日选题 — 适合我公众号写的热点话题

如果你做电商，可能是这样：

- 平台政策 — 淘宝、抖音、拼多多的规则变动
- 爆款速报 — 各平台跑出来的新爆品
- 工具更新 — 千牛、抖店等电商工具的新功能
- 行业数据 — GMV、投流、转化率相关
- 操盘手分享 — 有结果的卖家公开分享的经验

板块不要超过5个，每个板块3-5条。太多没人看。

**定搜索关键词。** 每个板块对应哪些搜索词？这直接决定日报搜到的内容质量。比如"模型速报"对应的搜索词是：AI大模型发布、LLM benchmark、开源模型更新。

## 第二步：写日报的Prompt

这一步最关键。好日报和垃圾日报的差距，全在prompt上。

打开OpenClaw的配置，创建一个Cron任务。任务的核心就是一段prompt，告诉AI怎么搜、怎么写。

我踩了很多坑之后，总结出三条必须写进prompt的铁律：

**铁律一：只能搜，不能编。**

prompt的最顶部写死这句话：

"所有信息必须来自搜索结果。搜不到就写"今日暂无"，绝对不编造任何新闻、数据或链接。没有真实来源链接的内容一律不写。"

为什么放最顶部？因为AI有个毛病——搜不到东西的时候，它会自己编一条看起来很像真的假新闻，链接点进去404。我有一次差点把一条"OpenAI发布GPT-6"的假消息转出去。从那以后，这条铁律雷打不动。

**铁律二：多轮搜索。**

一开始我每个板块只搜一次，信息经常不全。后来改成分组搜索——每个板块用2-3组不同的关键词各搜一轮，总共搜13轮。覆盖面一下就上来了。

而且必须限定时间范围：只搜24小时以内的内容。不加这个限制，AI会拿一个月前的旧闻来凑数。

**铁律三：每条带行业价值分析。**

纯新闻没有价值，百度能干。日报的核心是"这条信息跟我有什么关系"。

所以prompt里要求每条信息后面必须跟一段分析：这件事对从业者意味着什么，该关注还是该忽略，有没有行动建议。

对比一下：

没有分析的日报：
"Claude发布企业插件。链接。"
看完等于没看。

有分析的日报：
"Claude发布10个企业级插件，打通金融、法务、HR工作流。行业价值：Claude从聊天工具正式转型企业操作系统，做B端AI方案的团队要重点关注插件架构思路。链接。"

后者才是老板（或者你自己）真正需要的东西。

## 第三步：创建定时任务

OpenClaw用命令行管理定时任务。打开终端，执行：

```
openclaw cron add \
  --name "AI日报" \
  --cron "0 22 * * *" \
  --tz "UTC" \
  --session isolated \
  --timeout-seconds 480 \
  --announce \
  --message "你的日报prompt写在这里"
```

几个参数解释一下：

- `--cron "0 22 * * *"` — UTC时间22点执行，也就是北京时间早上6点
- `--session isolated` — 独立会话运行，不会和你的日常对话混在一起
- `--timeout-seconds 480` — 超时8分钟。日报要做十几轮搜索，5分钟不够，8分钟比较稳
- `--announce` — 执行完自动把结果推送给你

设好之后可以手动跑一次测试：

```
openclaw cron run --id 你的任务ID
```

看看生成的日报质量怎么样。不满意就改prompt，反复调，直到你觉得"这份日报我每天愿意花5分钟看"为止。

## 第四步：调到满意为止

第一版日报大概率不会让你满意。我的日报调了不下10次。

常见问题和解法：

**问题：搜到的信息太少。**
加搜索轮次，同一个板块用不同关键词多搜几次。

**问题：信息太多太杂。**
在prompt里写死"每板块不超过5条，超出的只留最有价值的"。

**问题：分析太泛，像废话。**
在prompt里给个示范格式，让AI照着写。比如："分析要具体到：影响谁、该怎么做、优先级高不高。"

**问题：搜到过期内容。**
确保搜索限定了24小时时间窗口。OpenClaw的Brave Search默认支持时效过滤。

**问题：偶尔还是会编造。**
在铁律那段话前面加"这是最高优先级规则"。AI对"最高优先级"这类措辞比较敏感。

每次调完prompt，用 `openclaw cron run` 手动跑一次验证效果。别怕麻烦，调好一次，以后每天自动受益。

## 通用模板：直接复制

不管你是什么行业，把【】里的内容换成你的就能用：

```
你是一个资深的【行业名称】行业分析师。
每天生成一份行业日报。

最高优先级规则：所有信息必须来自真实搜索结果。
搜不到就写"今日暂无"，绝对不编造任何新闻、数据或链接。
没有真实来源链接的内容一律不写。

板块设置：
1.【板块1名称】— 搜索关键词：【词A】【词B】【词C】，3-5条
2.【板块2名称】— 搜索关键词：【词A】【词B】，3-5条
3.【板块3名称】— 搜索关键词：【词A】【词B】，2-3条
4.【板块4名称】— 搜索关键词：【词A】【词B】，2-3条
5.【板块5名称】— 搜索关键词：【词A】【词B】，2-3条

搜索要求：
- 每个板块用2-3组不同关键词分别搜索
- 只搜24小时以内的内容
- 每板块不超过5条，不够不凑，超出只留最有价值的

输出格式：
每条信息包含三部分：
1. 事件：发生了什么（一句话说清楚）
2. 行业价值：跟从业者有什么关系，该怎么应对
3. 来源链接

标题格式：【行业名称】日报 · 日期
```

复制这段 — 改关键词 — 用 `openclaw cron add` 创建定时任务 — 手动跑一次测试 — 调到满意 — 完事。

以后每天自动跑，你只管看。
